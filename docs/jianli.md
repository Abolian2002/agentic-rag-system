传统做法每次启动应用时，即使文档没变，也要把所有切块重新向量化并存入向量库——这一步很耗时（调用 embedding 模型）。增量索引的做法是给每个文档算一个 Hash 指纹，存起来。下次启动时对比指纹——如果指纹没变，说明文档没改过，跳过不处理；如果指纹变了，才重新向量化并存库。这样就只处理有变化的文档，而不是每次都全量重建向量索引。

### . SHA-256 Hash 是什么？
Hash（哈希/摘要） = 把任意长度的文件内容压缩成一个固定长度的"指纹字符串"。
比如一个 PDF 文件，内容可能有几 MB，但它的 SHA-256 Hash 只是一个 64 字符的十六进制串：

- 同一个文件 → 同一个 Hash（不变）
- 文件改了一个字 → Hash 完全变了
- 不同文件 → Hash 不同（几乎不会碰撞）

### . "热更新"是什么？
热更新 = 不重启应用，就能让新文档生效。
传统做法：新增文档后，重启应用 → 全量重建 → 才能用。
Hash 增量索引 + 热更新 ：

- 新增文档 → 算 Hash → 发现是新文件 → 立刻索引 → 立刻能用
- 不需要重启，不需要全量重建
